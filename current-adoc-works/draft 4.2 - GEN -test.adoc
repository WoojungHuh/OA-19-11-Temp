|ID |GEN-001
|TC-11 Version |V1.0 & Draft (1.3)

|V1.0 Requirement Text |The Ethernet Switch shall implement all "5.1
Required Capabilities" except "f) RSTP“ and “g) BPDUs” of the IEEE
802.1D Standard [1].

|Draft Title |REQ_ETH_SWITCH_V2_L2_G_001 Conformance to IEEE 802.1Q
bridging requirements

|Draft Requirement Text |The Ethernet Switch shall implement all MAC
Bridge component requirements of [1], 5.13, except f) (RSTP) and g)
(BPDUs).

|Config Type & Attribute|
* feature

|Generic config |

|YANG config a|
[source,yang]
----
module: ieee802-dot1q-bridge
	+--rw bridges
	    +--rw bridge* [name]
	    ...
	    +--rw component* [name]
	    ...
	        +--ro bridge-port* if:interface-ref
	        +--ro capabilities
	        ...
	        +--rw filtering-database
	        ...
	        +--rw permanent-database
	...
augment /if:interfaces/if:interface:
	+--rw bridge-port
	...
----

Specify the following performance characteristics of the implementation
(buffer(s) for input and output) in the documentation of the Ethernet
Switch:

- Maximum Frame Buffer Size (overall) [bytes]

- Maximum Frame Buffer Size depending on frame size in bytes [number of
frames]. This could be a function or a table.

- Configuration options to distribute and limit the buffer size per port
and CoS queue.

- Maximum Frame Buffer Size (overall) [bytes], +

frames]. This could be a function or a table, +
- Configuration options to distribute and limit the buffer size per port
and QoS queue

|ID |GEN-013A
|TC-11 Version |V1.0 & Draft (1.3)

|V1.0 Requirement Text |The Ethernet Switch shall support mirroring for
each port. This includes mirroring frames from multiple ports to a
single port with filtering depending on Port, SA/DA MAC Addresses and
VLAN membership. This includes local and remote mirroring. It shall be
possible for remote mirroring to change the VLAN-ID. At least 16 rules
shall be supported, each being applicable to an arbitrary subset of the
overall ports (e.g. by using a port bitmask).

|Draft Title |REQ_ETH_SWITCH_V2_L2_G_010 Port mirroring

|Draft Requirement Text |The Ethernet Switch shall support port
mirroring such that traffic from the configurable ingress ports can be
duplicated on to a designated destination port.

|ID |GEN-013B
|TC-11 Version |Draft (1.3)

|V1.0 Requirement Text |N/A

|Draft Title |REQ_ETH_SWITCH_V2_L2_G_011 Dedicated VLAN for mirrored
traffic

|Draft Requirement Text a|
The Ethernet Switch shall support at least one of the following two
methods to put mirrored traffic from configurable ingress ports in a
configurable VLAN on to a designated destination port:

1.Replacing the VID (or for untagged frames the assigned PVID) with a
configurable mirroring VID.

2.Adding an additional outer VLAN tag with TPID 0x8100; with this
method, the suppression of frames with two TPID 0x8100 tags needs to be
disabled as described in requirement VLAN-009B
